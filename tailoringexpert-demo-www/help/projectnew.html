New Project Wizzard

This is the wizzard to create a new project. Wizzard is performed in four steps:

  1. select catalog version
  2. upload a screeningsheet
  3. check calculated selection vector based on screeningsheet input and update vector if needed
  4. create project

Select catalog version

This is the first step in creating a new project. If only one catalogue is valid at the time of creation, it will be selected. Any notes entered will be added to the initial tailoring created for the project. They are not mandatory.

Upload a screeningsheet

A screening sheet is required as input for project creation and tailoring. Format and content are tenant-specific. Once uploaded, extracted values are displayed.

Check calculated selection vector

A selection vector is calculated based on the given screen sheet. A selection vector defines levels for all parameters of a tailoring. These values are used to select requirements for the initial tailoring. The values can be either modified by

  • selecting a profile
  • adjusting the values by clicking on them

Predefined Profiles

Pragmatic LEO Mission

These are mainly earth observation missions, which allow for pragmatic approaches, e.g. NewSpace or COTs application. Examples for such missions are TerraSar, Tandem X or EnMap.

Operational LEO Mission

These are mainly earth observation missions, which do not allow for pragmatic approaches since they are requiring operational results and availability. Examples for such missions are the Copernicus-Sentinel Satellites.

Platform Equipment

This is equipment, which will be flying on a satellite platform for either an experiment, payload or the platform service. Examples for such equipment are the standard equipment from suppliers like TESAT or any experiment/payload equipment.

Telecom

These are telecommunication satellites. Examples for such missions are the EDRS-Satellites.

ISS-Experiment außen

These are payloads or experiments that will be placed on any outside platform of the ISS. The dedicated platform provider is to be distinguished in the screening-sheet and tailoring of requirements.

ISS-Experiment intern

These are payloads or experiments or service equipment (e.g. on life support) that will be placed inside of the ISS. The dedicated module in which the experiment will be conducted is to be distinguished in the screening-sheet and tailoring of requirements.

Bodenaktivitäten

This coves all ground based activities, from ground experiments to facilities supporting operation or testing.

"Studentenmission" (Nano/Micro-Mission) & Technologiedemonstration

These are mainly CubeSat or Small Sat activities, which are part of student education or technology demonstration.

Deep Space Science Mission / Instrument oder Lander

These are all missions, which are supposed to explore e.g. the Moon, Mars, Venus, Jupiter, Planetary Moons or scientific mission to Lagrange points respectively deep space. This missions can be full missions, Lander modules or experiments/instrument payloads on these.

Create project

By clicking the Create button, the project is created with an initial tailoring using the defined selection vector. for the (pre-)selection of requirements.
